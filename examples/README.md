# サンプルデータ（すべて架空です）

このフォルダはスキルの動作を確認するための**完全に合成されたサンプルデータ**です。実在の届出番号・物件・宿泊者・予約は一切含まれません。

## ファイル一覧

- `sample-airbnb.csv` — Airbnb ホスト画面からエクスポートされる予約CSV（13列・英語ロケール）の形を再現した見本
- `sample-registry.xlsx` — 同梱テンプレート（`minpaku-teiki-hokoku/assets/guest-registry-template.xlsx`）に沿った宿泊者名簿の記入例
- `minpaku-config.yaml` と `2026-04_2026-05/`（normalized_reservations.csv・kujo-log.csv） — そのまま計算エンジンに渡せる作業フォルダの見本
- `sample-output/` — 下記コマンドで生成した帳票一式のスナップショット

## 実行コマンド

リポジトリ直下で:

```
python3 minpaku-teiki-hokoku/scripts/generate_report.py examples 2026-04_2026-05
```

`examples/2026-04_2026-05/output/` に帳票一式が生成されます（このパスは .gitignore 済み。コミット対象のスナップショットは `sample-output/` です）。

## 意図的に仕込んである警告（バグではありません）

検証機能のデモとして、次のケースをデータに意図的に含めてあります。検証レポート（`validation_2026-04_2026-05.md`）に警告が出るのが**正常な動作**です。

- 複数OTAからの二重取り込み（Airbnb と Booking.com の同一滞在）
- ステータス不明の予約
- 期間をまたぐ滞在
- 期間外の苦情記録
- 宿泊実績0件の物件（0件報告）

- **二重取り込みの2名分が集計に含まれたままなのも正常です。** エンジンは疑わしい重複を黙って消さず、警告してユーザーの判断に委ねる設計だからです。実際の利用では、警告に従って重複行を1行に統合してから提出してください。
