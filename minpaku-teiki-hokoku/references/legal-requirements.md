# 定期報告の法的要件（住宅宿泊事業法 第14条）

この文書は、住宅宿泊事業（民泊新法）の定期報告に関する **公式ルールの根拠集** です。すべての記述に公式ソースのURLを付けています。本スキルはデータの「準備」を行うものであり、**報告の提出とその内容に対する責任は住宅宿泊事業者（届出者）本人にあります**。

最終確認日: 2026-07-02。法令・ガイドラインは改正されることがあります。判断に迷う場合は必ず原典と所管の自治体窓口で確認してください。

---

## 1. 報告義務の根拠

**住宅宿泊事業法 第14条**（原文）:

> （都道府県知事への定期報告）第十四条 住宅宿泊事業者は、届出住宅に人を宿泊させた日数その他の国土交通省令・厚生労働省令で定める事項について、国土交通省令・厚生労働省令で定めるところにより、定期的に、都道府県知事に報告しなければならない。

出典: https://laws.e-gov.go.jp/law/429AC0000000065

**住宅宿泊事業法施行規則 第12条**（原文）:

> 第十二条 法第十四条の国土交通省令・厚生労働省令で定める事項は、次に掲げるものとする。
> 一 届出住宅に人を宿泊させた日数 / 二 宿泊者数 / 三 延べ宿泊者数 / 四 国籍別の宿泊者数の内訳
> ２ 住宅宿泊事業者は、届出住宅ごとに、毎年二月、四月、六月、八月、十月及び十二月の十五日までに、それぞれの月の前二月における前項各号に掲げる事項を、都道府県知事に報告しなければならない。

出典: https://laws.e-gov.go.jp/law/429M60000900002

要点:
- 報告単位は **届出住宅ごと**（届出番号ごと）。事業者単位でも OTA アカウント単位でもない。5物件持つ管理者は毎回5件の報告を出す。
- 報告先は名目上「都道府県知事」だが、法第68条により保健所設置市・特別区ではその市長・区長が処理する（例: 大阪市長、東京の区長）。民泊制度運営システム経由なら宛先の振り分けはシステム側が行う。

## 3. どの物件が報告対象か（物件タイプの判定）

### 3.1 住宅宿泊事業の届出住宅（M番号）→ 定期報告の対象

- 届出番号の表示形式は「第M＋9桁数字＋号」。数字の先頭2桁は JIS 都道府県コード（13=東京都、26=京都府、27=大阪府 …）。
- 判定用の正規表現: `^(第)?M\d{9}(号)?$`（「第」「号」の有無は表記ゆれとして正規化する）。
- 例（架空）: 第M130000000号（東京都）、第M260000000号（京都府）。
- 出典（形式の確認元）: 東京都の届出住宅一覧 https://www.sangyo-rodo.metro.tokyo.lg.jp/documents/d/sangyo-rodo/tokyo070331_pdf_todokedezyun / 京都市の届出住宅一覧（全件 M26 で始まる） https://minpakuportal.city.kyoto.lg.jp/wp-content/uploads/2026/06/080531todokedeichiran.pdf
- 注意: 数字3〜9桁目が連番以外の意味を持つかは未確認（東京都・京都市の2都市のサンプルからの推定）。

### 3.2 旅館業許可（ホテル・旅館・簡易宿所）→ 定期報告の対象外

- 旅館業には国への定期的な実績報告制度は存在しない（公式ソースで該当制度が確認できない。不存在の完全証明は不可能だが、厚労省資料に言及なし）。
- 継続義務は宿泊者名簿等。2023-12-13 施行の旅館業法施行規則改正で名簿の記載事項は「氏名・住所・連絡先」（職業は削除、連絡先が追加）＋日本国内に住所を有しない外国人は国籍・旅券番号。保存3年。
- 出典: https://www.mhlw.go.jp/web/t_doc?dataId=79090000&dataType=0&pageNo=1 / https://www.pref.okayama.jp/page/detail-17753.html
- 許可番号に全国統一形式はなく、自治体の「指令」様式の文字列（例:「〇〇市指令 保生 第〇〇号」）が多い（二次情報。判定は正規表現でなくユーザーへの確認で行うこと）。

### 3.3 特区民泊（国家戦略特別区域外国人滞在施設経営事業）→ 定期報告の対象外

- カレンダー式の定期報告はない。報告は **求めに応じた報告徴収**（国家戦略特別区域法第13条第9項）と、自治体によっては **事象発生時の報告**（大田区: 近隣からの苦情、常駐体制の変更、火災等の避難事態 → 様式別紙2で「速やかに報告」）。
- 出典: 大阪市特区民泊ガイドライン https://www.city.osaka.lg.jp/kenko/cmsfiles/contents/0000341/341012/20211101gaidorain.pdf / 大田区ガイドライン https://www.city.ota.tokyo.jp/kuseijoho/kokkasenryakutokku/ota_tokkuminpaku.files/20260406tokkuguideline.pdf
- 継続義務: 滞在者名簿（職業を含む・国籍・旅券番号）3年保存、外国人の旅券写しの保管義務、チェックイン/アウト双方の本人確認、最低2泊3日 など。
- 認定番号の形式は自治体ごと（例: 「大田区特区民泊 第〇〇号」— 二次情報）。M番号に一致しない番号は推測せず、ユーザーに届出済証等の確認を求めること。

### 3.4 イベント民泊 → 対象外

- 年数回・1回2〜3日程度、宿泊施設不足（または2019年改正でホームステイ交流目的）、自治体の要請等の公共性、の要件下で旅館業法の適用外。定期報告なし。
- 出典: https://www.mhlw.go.jp/file/06-Seisakujouhou-11130500-Shokuhinanzenbu/0000171350.pdf / https://www.mlit.go.jp/common/001264322.pdf

### 3.5 混在ケース（重要）

同一物件に特区民泊の認定や旅館業の許可を取っても、**住宅宿泊事業の届出を取り下げない限り第14条の報告義務は続き、特区民泊としての宿泊を含む全宿泊実績を報告する**（180日制限も併存）。報告義務を終わらせるには廃業届が必要。

- 出典: 大阪市「定期報告の手引き」 https://www.city.osaka.lg.jp/keizaisenryaku/cmsfiles/contents/0000504/504602/tebiki.pdf （自治体解釈。国の文書での明示は未確認）

判定に迷ったら: 届出済証を確認するか、自治体が公表する届出住宅一覧と照合する。**推測で分類しない。**

## 4. 4つの報告項目の公式定義と計算例

出典（共通）: 住宅宿泊事業法施行要領（ガイドライン）2-2⑼②（最終改正 令和6年12月24日） https://www.mlit.go.jp/kankocho/minpaku/content/001622384.pdf / 観光庁「定期報告に係る留意事項」 https://www.mlit.go.jp/kankocho/content/001247661.pdf / 大阪市手引き（計算例） https://www.city.osaka.lg.jp/keizaisenryaku/cmsfiles/contents/0000504/504602/tebiki.pdf

### 4.1 ① 届出住宅に人を宿泊させた日数（宿泊日数）

- 法第2条第3項＋施行規則第3条の算定方法（180日制限と同じ数え方）。規則第3条（原文）:「…正午から翌日の正午までの期間を一日とする。」
- **公式の計算例**（留意事項）: 6月20日17時チェックイン → 6月24日10時チェックアウト = **4日**（泊数と一致: 6/20, 6/21, 6/22, 6/23 の4泊）。
- **届出住宅ごとにユニークな日付を数える**: 同じ日に複数グループ（または同一届出番号内の複数部屋・複数リスティング）が重なっても、その日は **1日**（大阪市手引きの例: 7/2 が2グループで重複しても1日と数え、合計9日）。
- したがって、1つの届出番号を複数の OTA リスティングとして運用している場合、宿泊日数は **全リスティング横断の「宿泊があった日付の和集合」** で数える。
- ノーショー（当日キャンセルで実際に宿泊しなかった場合）や、真正な賃貸借契約による概ね1か月以上の滞在は算入しない（令和6年12月24日改正のガイドライン 2-2⑵①。ただし都道府県知事等が事実関係を認める場合であり、宿泊契約を事後的に賃貸借契約に読み替えることは不可）。本スキルのエンジンは status=cancelled の予約を集計から除外する。

### 4.2 ② 宿泊者数

- ガイドライン:「実際に届出住宅に宿泊した宿泊者の総数」。
- 同一人物の連続した宿泊は1人。同一人物が期間を空けて複数回宿泊した場合は回数分数える（留意事項）。
- **乳幼児を除外する公式規定は存在しない。** ガイドライン・FAQ集・留意事項・定期報告QA・大阪市手引きの全文検索で「乳幼児」への言及はゼロ（2026-07-02 確認）。定義は文字どおり「実際に宿泊した宿泊者の総数」であり、大阪市の計算例も全員を一律に数える。**本スキルは大人＋子ども＋乳幼児の全員を数える**（公式定義の文言どおり）。虚偽の報告は罰則の対象になるため、根拠のない除外を行ってはならない。
- **期間跨ぎルール（straddle）**: 宿泊者数は報告期間ごとに数える。2つの報告期間にまたがる連続宿泊は **それぞれの期間で1人ずつ** 数える。公式FAQの例: 3月末〜4月初にまたがる宿泊 → 4月15日期限の報告（2・3月分）に1人、かつ6月15日期限の報告（4・5月分）にも1人。
  - 出典: FAQ集 ⑤Q4 https://www.mlit.go.jp/kankocho/minpaku/content/001396589.pdf

### 4.3 ③ 延べ宿泊者数（延べ人数）

- ガイドライン（原文）:「実際に届出住宅に宿泊した宿泊者について、１日宿泊するごとに１人と算定した数値の合計をいう。例えば、宿泊者１人が３日宿泊した場合は３人となる。」
- 実質的に **人数 × 泊数**（泊数は報告期間内に切り詰める）。留意事項の例: 3人×2泊（6人）＋5人×6泊（30人）= **36人**。
- 期間をまたぐ宿泊は、各期間に属する泊数分だけ各期間に計上する（大阪市手引き）。
- 民泊制度運営システムの Web 画面ではこの項目だけ自動計算されず手入力（references/portal-guide.md 参照）。

### 4.4 ④ 国籍別の宿泊者数の内訳

- ガイドライン:「『宿泊者数』の国籍別の内訳をいう」。留意事項はさらに「日本国内に住所を有しない宿泊者の国籍の内訳」とする。
- **公式の区分は次の22カテゴリ（この順序で固定・数値入力のみ・自由記述なし）**:

  日本 / 韓国 / 台湾 / 香港 / 中国 / タイ / シンガポール / マレーシア / インドネシア / フィリピン / ベトナム / インド / 英国 / ドイツ / フランス / イタリア / スペイン / ロシア / 米国 / カナダ / オーストラリア / その他

  - 出典: 民泊制度ポータルサイト「電子宿泊者名簿・定期報告方法」（22区分を列挙） https://www.mlit.go.jp/kankocho/minpaku/business/system/regular_report.html / 操作手順書の画面スクリーンショット 図2-43・図2-46 https://www.mlit.go.jp/kankocho/minpaku/business/system/content/001596131.pdf
  - **注意**: 操作手順書の本文表（表2-7）は「香港」を列挙から落としているが、これは文書上の誤記。同じ手順書のスクリーンショットとポータルページはいずれも香港を含む22区分。**香港を含めること。**
- **住所ベースの公式ルール**（宿泊旅行統計調査の方法に整合。決定的ルールなので推測ではなくコードで適用する）:
  - 日本国内に住所を有する外国人 → **「日本」** に計上（FAQ集 ⑤Q5）。
  - 日本国内に住所を有しない日本人 → **「その他」** に計上（FAQ集 ⑤Q6）。
- 多国籍グループは各人を各自の国籍で数える。
- **国籍不明の宿泊者の扱いに公式ルールは存在しない。** 「その他」への振り分けや氏名からの推定は実務上の慣行であり、法令上の規定ではない。本スキルは推定した値に必ず出所（名簿/フォーム/パスポート/住所ルール/推定/その他）を表示し、事業者本人の確認を経て確定する。判断がつかない場合は自治体窓口への確認が確実（大阪市は観光担当への電話を案内）。
- 整合性チェック: **22区分の合計 = 宿泊者数** でなければならない（Web 画面でも国籍別入力の合計が宿泊者数として自動計算される）。

## 5. ゼロ報告（宿泊実績がない期間）

- 宿泊が0日の期間でも報告は必要。ガイドライン 2-2⑼④（原文):「賃貸中、休業中等の理由で日数が０日であっても定期報告を行う必要がある。」宿泊日数・宿泊者数・延べ人数に0を入れて保存する（定期報告QA #3）。
- **届出が受理された日から義務が始まる**。営業開始前でも受理日以降の期間は0件報告が必要（例: 5月20日受理・6月1日営業開始 → 5月20日〜31日を含む期間のゼロ報告が必要）。
- 住宅宿泊事業法に「休業届」の制度はない。長期間実施見込みがないなら廃業等届出書を検討する（京都市手引き・大阪市手引き）。
- 報告しない場合、都道府県知事等は督促を行い、連絡が取れず事業実態が確認できないときは確認から30日後に事業廃止とみなされ得る（ガイドライン 2-2⑼④）。
- 出典: https://www.mlit.go.jp/kankocho/minpaku/content/001622384.pdf / https://www.mlit.go.jp/kankocho/content/001247862.pdf

## 6. 180日ルールとの関係

- 法第2条第3項: 住宅宿泊事業は「人を宿泊させる日数…が一年間で百八十日を超えないもの」。
- 規則第3条（原文）:「毎年四月一日正午から翌年四月一日正午までの期間において人を宿泊させた日数とする。この場合において、正午から翌日の正午までの期間を一日とする。」→ **年度は4月1日正午起点**。報告期間（暦月2か月）とは区切りが一致しない点に注意（システムの報告期間表記も「20XX年度XX月～XX月」）。
- 定期報告の「宿泊日数」は180日制限と同じ算定方法なので、年6回の報告がそのまま公式の180日カウント記録になる。
- 届出住宅単位で数え、年度内の事業者変更があっても通算される。実際に宿泊料を得て宿泊させた日のみ数える（掲載日・入金日ではない）。
- 超過して宿泊させると旅館業法第3条第1項違反となる（ガイドライン 2-3⑴①）。
- 民泊制度運営システムは年度内累計宿泊日数が120〜169日で注意喚起メール、170日以上で「要注意」メールを送る（操作手順書 第2章第4節）。本スキルのエンジンも120日/170日で警告を出す。
- 出典: https://laws.e-gov.go.jp/law/429M60000900002 / https://www.mlit.go.jp/kankocho/minpaku/content/001622384.pdf

## 7. 罰則（事実として1回だけ記載）

**法第76条**（原文）:「次の各号のいずれかに該当する者は、三十万円以下の罰金に処する。… 三 第十四条の規定による報告をせず、又は虚偽の報告をした者」— 定期報告の不提出・虚偽報告は **30万円以下の罰金**（e-Gov の法文XMLで条番号まで確認済み）。行政措置としては業務改善命令（法第15条）、業務停止・廃止命令（法第16条）もある。

出典: https://laws.e-gov.go.jp/law/429AC0000000065

## 8. 宿泊者名簿の義務（報告データの法定の源泉）

- **法第8条**: 名簿は届出住宅または事業者の営業所・事務所に備え付け、都道府県知事の要求があれば提出する。宿泊者には記載事項の申告義務がある（第8条第2項）。
- **施行規則第7条**: 記載事項は「氏名、住所、職業及び宿泊日のほか、宿泊者が日本国内に住所を有しない外国人であるときは、その国籍及び旅券番号」。**作成の日から3年間保存**。
  - 注: 旅館業は2023年改正で「職業」を削除したが、**民泊（住宅宿泊事業）の名簿には職業が今も必要**（2026年4月版の京都市手引きでも確認）。
- 本人確認（ガイドライン 2-2⑷①）: 宿泊開始前に全宿泊者について、対面 または (A)顔と旅券が画像で鮮明に確認でき (B)映像が届出住宅内・近傍または営業所から発信される ICT で行う。
- 日本国内に住所を有しない外国人: 国籍・旅券番号を記載し、旅券の提示を求め、**旅券の写しを名簿とともに保存する**（写しの保存で氏名・国籍・旅券番号の記載に代えられる）。説明しても提示を拒否された場合は「最寄りの警察署に連絡する等適切な対応」。
- 名簿は**宿泊者全員**を記載する（代表者のみは不可）。
- 警察からの求めに応じた名簿の提供は宿泊者の同意なく適法（ガイドラインが個人情報保護法の第三者提供の例外規定を引用。ガイドライン本文は改正前の条番号 23条1項4号 を引くが、現行法では 27条1項4号 に相当）。
- **スキル設計上の含意**: 名簿の国籍欄が法定必須なのは「日本国内に住所を有しない外国人」のみ。日本人と日本在住の外国人には国籍欄の法定義務がない。したがって報告の国籍別内訳には一定の推定・振り分けが構造的に避けられない — ただし非居住外国人については法律上すでに国籍＋旅券番号＋旅券写しを保有しているはずであり、まずそれを参照する。
- 出典: https://laws.e-gov.go.jp/law/429M60000900002 / https://www.mlit.go.jp/kankocho/minpaku/content/001622384.pdf

## 9. 誰が報告するのか（義務は委任できない）

- 定期報告の義務は **住宅宿泊事業者（届出者）本人** にある。住宅宿泊管理業者の法定の「定期報告」（法第40条）は **事業者に対する** 報告であり、行政への報告ではない。
- 京都市手引き（原文）:「定期報告は住宅宿泊事業者（届出者）の義務です。住宅宿泊管理業者が行うことはできません。」
- 民泊制度運営システム上も、管理業者ロールのアカウントに定期報告機能は存在しない（管理業者向け操作手順書に定期報告の記載なし: https://www.mlit.go.jp/kankocho/minpaku/business/system/content/001596086.pdf ）。
- 事務作業を第三者（行政書士・管理会社）が手伝う場合も、観光庁の公式文書「代理人による民泊制度運営システムの利用について」により **届出者本人に発行されたIDでログインして行う**（代理人のメールアドレスをIDにはできない）: https://www.mlit.go.jp/kankocho/content/001225372.pdf
- **本スキルの立場**: スキルは報告データを準備するだけであり、提出は事業者本人が行い、内容の責任も事業者本人が負う。スキルを「代行サービス」として扱ってはならない。

## 10. 自治体の上乗せルール（「国の4項目だけで足りる」とは限らない）

- **京都市は定期報告そのものに項目を追加している**（条例第16条第1項）: 国の4項目に加えて **苦情の件数・苦情を受けた日時・苦情の内容・苦情への対応状況**、さらに廃棄物を処理した後最初の報告時には **ごみ処理関係書類** の添付が必要。詳細は `references/kyoto.md`。
- 千代田区など、条例に基づく追加記録（廃棄物・衛生管理・管理者駐在・苦情記録等）を求める自治体もある: https://www.city.chiyoda.lg.jp/koho/kurashi/jutakushukuhakujigyo/hokoku.html
- 東京23区の多くには営業できる区域・期間の上乗せ条例があるが、これは報告様式ではなく営業条件の話（二次情報）。
- **47都道府県の網羅調査は行っていない。** 京都市（追加あり）と大阪市（追加なし・国の様式と同内容）以外の自治体については、必ず所管窓口と次の公式ディレクトリで確認すること:
  - **国土交通省 民泊制度ポータル「各自治体の情報・窓口案内」**: https://www.mlit.go.jp/kankocho/minpaku/municipality.html
- 宿泊税（京都市・大阪府市・東京都など）は定期報告とは別の義務であり、期限も一致しない（京都市手引き ※2）。本スキルの対象外。
