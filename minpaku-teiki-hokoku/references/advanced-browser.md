# 【上級者向け付録】ブラウザ経由で Airbnb の全予約履歴を一括取得する

> **⚠️ この付録は任意（オプション）かつ上級者向けです。**
>
> - **Claude Cowork（仮想マシン環境）の中では動作しません。** Cowork のサンドボックスからは airbnb.com へのネットワークアクセスも、ログイン済みブラウザへの接続もできません。**Cowork をお使いの方はこの付録を丸ごとスキップ**し、通常のページごとCSVエクスポート（`ota-export-guides.md` §1.5–1.6）を使ってください。
> - ここで使うのは Airbnb の**非公開・非公認エンドポイント**です。予告なく変更・停止される可能性があり、動かなくなっても通常のエクスポート手順が常にフォールバックです。
> - **自分自身のアカウント**に対してのみ行ってください。

## 何が解決できるか

Airbnb の画面上の「CSVファイルをダウンロード」は表示中の1ページ（約40行）しか出力しません（`ota-export-guides.md` §1.6）。予約件数が多いと、ページ送りでのダウンロードは手間です。

実は、そのエクスポートボタンが裏で呼んでいる認証付きエンドポイントを、**自分のログイン済みブラウザから**ページ番号を変えながら繰り返し呼ぶことで、同じ13列CSVを全履歴分取得できます（2026年6月時点の実測挙動 ⚠️ ただし非公認のため将来の保証なし）。

## エンドポイントの仕様（観測ベース）

```
GET https://www.airbnb.com/api/v2/download_reservations
    ?_format=for_remy
    &_limit=40
    &_offset=0
    &collection_strategy=for_reservations_list
    &sort_field=start_date
    &sort_order=desc
    &status=accepted%2Crequest%2Ccanceled
    &key=<自分のブラウザが使っている公開Webキー>
    &currency=JPY
    &locale=ja
```

- 認証はブラウザの**セッションCookie**で行われます（＝ログイン済みの自分のブラウザ内でのみ機能します）。
- `_limit` はサーバー側で **40 に固定（clamp）** されます。40より大きい値を指定しても無視されます。
- `_offset` を `0, 40, 80, 120, …` と増やしながら繰り返し取得し、**データ行が空になったら終了**します。
- `locale` がCSVヘッダーの言語を決めます（`en` = 英語ヘッダー、`ja` = 日本語ヘッダー）。`currency` は Earnings/収入列の通貨。
- `key` はAirbnbのWebページ自身が使っている公開APIキーで、後述のとおり**自分のブラウザの通信記録からコピー**します。どこかで調べたり他人から入手したりするものではありません。

## 方法A: ブラウザ操作型のAIアシスタント経由（例: Claude in Chrome）

自分のログイン済みChromeに接続できるブラウザ操作型アシスタント（Claude in Chrome など）を使っている場合は、次のように依頼できます。

1. 自分のブラウザで Airbnb にログインした状態で、`airbnb.com/hosting/reservations` を開く。
2. アシスタントに依頼する（例）:「このページのエクスポートが呼んでいる `api/v2/download_reservations` を、`_offset` を40ずつ増やしながらデータ行が空になるまで取得し、ヘッダー行は1回だけ残して結合したCSVを `reservations_full.csv` として保存して」。
3. できあがったCSVを作業フォルダに入れ、以降は通常のフロー（列検証→正規化→集計）へ。

アシスタントはあくまで**あなたのブラウザ・あなたのセッション**の中で動きます。リクエストは順番に（並列にせず）実行させてください。

## 方法B: 手動（開発者ツール、DevTools）

1. Chrome等で Airbnb にログインし、`airbnb.com/hosting/reservations` を開く。
2. 開発者ツールを開き（Mac: `⌥⌘I` / Windows: `F12`）、**Network（ネットワーク）**タブを表示。
3. 画面上で「エクスポート」→「CSVファイルをダウンロード」を1回実行し、Networkタブに現れる `download_reservations` のリクエストを見つける。
4. そのリクエストを右クリック → **Copy → Copy as fetch**（または Copy as cURL）。これで正しいURL・`key`・Cookie を含んだ完全な呼び出しが手に入ります。
5. コピーした呼び出しの `_offset=0` を `40`, `80`, `120`, … に書き換えながら繰り返し実行し（fetch なら開発者ツールの Console で、cURL ならターミナルで）、それぞれの結果をファイルに保存。**返ってくるデータ行が空になったら終了。**
6. 全ファイルを結合する（**ヘッダー行は先頭の1回だけ**残す）。結合は手作業でも、スキルにファイル一式を渡して依頼しても構いません。
7. **検算**: 結合後の行数が、画面上の予約件数の期待値と一致することを確認してから使ってください。

## セキュリティ上の注意（必読）

1. **自分のアカウントのみ。** 他人のアカウントやアクセス権のないデータに対して行ってはいけません。これはアクセス制御の回避ではなく、自分がブラウザ画面から見られるものと同じデータを自分のセッションで取得する行為に限定されます。
2. **Cookie・セッショントークン・`key` を絶対に他人と共有しない。** 「Copy as cURL/fetch」でコピーした文字列には**ログイン情報そのもの（Cookie）が含まれます**。チャットへの貼り付け、GitHub Issue への添付、スクリーンショット共有は厳禁です。使い終わったらコピーした文字列は破棄してください。また、**Copy as cURL をターミナルで実行した場合、Cookieを含むコマンドがシェル履歴（`~/.zsh_history` や `~/.bash_history` 等）に残ります**。実行後に履歴から該当行を削除するか、そもそもシェル履歴に残らない DevTools の Console で fetch を実行する方法を優先してください。
3. **非公認エンドポイントであることを忘れない。** `_format=for_remy` を含むこのAPIはAirbnbのどの公式文書にも載っていません。パラメータ・挙動・ページサイズ（40）はすべて観測ベースで、**予告なく変わり得ます**。動かない場合は深追いせず、通常のページごとエクスポートに戻ってください。
4. **礼儀正しく。** リクエストは順次・少量で。数百件程度の予約履歴なら十数回の呼び出しで済みます。
5. **利用規約はご自身で判断を。** 自動取得がAirbnbの利用規約とどう関係するかの解釈はユーザー自身の責任です。不安であれば、この付録を使わずページごとエクスポートを使ってください（結果は同じです）。
